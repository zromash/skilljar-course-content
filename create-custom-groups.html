Key Takeaways

 

Custom groups allow Hub administrators to create reusable groups of users that can be referenced across scheduling, messaging, and other Hub actions.

 

Groups are created in the Profiles module under the Custom Groups tab, and can span multiple teams within your organization.

 

Adding a user type to a group keeps membership dynamic — new users added to that type are automatically included, and removed users are automatically excluded.

 

View permissions control who can see the group — restrict to administrators only, or make it visible to all users.

Action Items

1

Navigate to Profiles > Custom Groups and select Add Custom Group in the top right corner.

2

Give the group a name, select the teams it should include, and configure view permissions.

3

Select Select Users and add members by individual or by user type, then select Save Group.

4

Use the three-dot icon next to any group to edit membership, view calendars, send a message, or delete the group.

Knowledge Check

Select each question to reveal the answer.

Where do you go to create a Custom Group in Hub?

Profiles → Custom Groups tab → Select Add Custom Group in the top right corner.

What happens when you add a user type to a custom group?

New users added to that type are automatically added to the group. Users removed from that type are automatically removed.

What does checking "All Team Users Only" do?

It restricts group visibility to administrators only. Uncheck it to allow all users to see the group.

Who can access the "All Teams" option when creating a group?

Only users with All Teams access AND the Manage Users and Groups permission.
